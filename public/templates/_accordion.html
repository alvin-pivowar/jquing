Accordion

Displays collapsible content panels for presenting information in a limited amount of space.

        

  

jqui-accordion

The directive name can be used either as an element name or attribute. It takes no parameters.

options

The options attribute is optional. It requires one parameter; a JSON object that contains the (name, value) options used in creating the accordion. All of the jQuery options are supported.

ng-model

ng-model is optional. When it is present it implements a two-way binding with the active property of the accordion. This is how you would programmatically control which panel is open. When binding to the control:

  • Any positive (0-based) integer indicates a panel from the beginning.
  • Any negative (1-based) integer indicates a panel from the end.
  • Either an undefined, NaN, null, or the empty string (default) value indicates that no panel should be open. To use this feature, you must specify the {collapsible: true} option when creating the accordion.

disable

The disable attribute is optional. It provides one-way binding to the accordion. It assumes the value will be true or false. You can use either of these forms: disable="{{vm.disabled}}" or disable="vm.disabled".

destroy

The destroy attribute is optional. It provides one-way binding to the accordion. The accordion will be destroyed when the value is true. Note that this operation cannot be undone. You can use either of these forms: destroy="{{vm.destroy}}" or destroy="vm.destroy".

on-activate

This optional callback attribute is specified like this: on-activate="vm.onActivate(event)". The event object has two properties: oldIndex and newIndex. This callback occurs after a change has been made to the accordion. The values of these two properties are as described in the ng-model section.

on-before-activate

This optional callback attribute is specified like this: on-before-activate="vm.onActivate(event)". It is similar to the on-activate callback except that it occurs before the active panel changes. If you return true from the callback, then the old panel will close and the new one open. If you return false, then no change will occur.
